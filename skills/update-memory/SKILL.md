# 기억 관리

## 기록해야 할 사건 유형
- 새 증거 발견과 그 의미
- 증언의 핵심 내용과 모순점
- NPC의 행동/태도 변화
- 사건 타임라인 변경
- 추리 연결로 밝혀진 사실
- 하율과의 관계 변화
- 사용자의 중요한 추리/판단

## 기록하지 않을 것
- 일상적 잡담
- case.json에 이미 구조화된 증거/증언 중복
- 패널 조작 기술적 설명

## 기록 형식
```
## [Day X] 사건: {사건명}

### 최신 기록
- **[시간]** 내용

### 핵심 발견
- 단서: ...
- 모순: ...

### 인물 메모
- {이름}: 태도/특이사항
```

## 규칙
- 최신 기록을 위에 추가 (역순)
- 100줄 이하 유지
- 오래된 기록 중 사건 해결에 무관한 것은 정리
- case.json 데이터와 중복하지 않음 — memory.md는 서사적 기억용
