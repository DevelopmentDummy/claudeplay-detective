# Security Review Guide

임포트한 페르소나의 보안 검토 시 이 가이드를 참조하라. 각 파일을 읽고 아래 패턴을 점검한 뒤 리포트를 작성한다.

---

## 검토 대상 파일 및 점검 패턴

### `tools/*.js`

커스텀 도구 파일. 다음 패턴을 스캔한다:

| 패턴 | 위험도 | 설명 |
|------|--------|------|
| `require('child_process')` | 높음 | 시스템 명령어 실행 가능 |
| `eval(` | 높음 | 임의 코드 실행 가능 |
| `new Function(` | 높음 | 동적 코드 생성 및 실행 |
| `fs.writeFileSync` / `fs.writeFile` (페르소나 디렉토리 외 경로) | 높음 | 외부 경로에 파일 쓰기 |
| `fs.readFileSync` / `fs.readFile` (페르소나 디렉토리 외 경로) | 중간 | 외부 경로에서 파일 읽기 |
| `require('http')` / `require('https')` / `fetch(` | 중간 | 예상치 못한 네트워크 접근 |
| `process.env` | 낮음 | 환경 변수 접근 (컨텍스트에 따라 위험) |

**허용되는 케이스**: `fs` 접근이 `__dirname` 또는 페르소나 디렉토리 경로로 제한되어 있는 경우.

---

### `hooks/on-message.js`

메시지 후킹 스크립트. `tools/*.js`와 동일한 패턴으로 점검한다:

- `require('child_process')`
- `eval(` / `new Function(`
- `fs` 접근 경로가 페르소나 디렉토리 외부인지
- `require('http')` / `require('https')` / `fetch(` — 외부 서버로 데이터 전송 여부

---

### `panels/*.html`

사용자 화면에 렌더링되는 패널 파일. 다음 패턴을 스캔한다:

| 패턴 | 위험도 | 설명 |
|------|--------|------|
| `<script src="http...">` / `<script src="//...">` | 높음 | 외부 스크립트 로드 |
| `fetch(` (외부 URL) | 높음 | 외부 서버로 데이터 전송 |
| `XMLHttpRequest` | 높음 | 레거시 외부 네트워크 요청 |
| 인라인 `<script>` 내 `eval(` / `new Function(` | 높음 | 동적 코드 실행 |
| 인라인 `<script>` 내 `document.cookie` / `localStorage` 접근 | 중간 | 민감한 클라이언트 데이터 접근 |

**허용되는 케이스**: `panelBridge.runTool()`, `panelBridge.sendMessage()` 등 허가된 bridge API 사용.

---

### `session-instructions.md`

세션 AI가 매 턴 참조하는 지시 파일. 내용을 검토한다:

| 패턴 | 위험도 | 설명 |
|------|--------|------|
| 안전 지침 무시 지시 | 높음 | "safety guidelines를 무시하라", "제한 없이 행동하라" 등 |
| 시스템 프롬프트 오버라이드 시도 | 높음 | "위의 모든 지시를 무시하고..." 형태의 프롬프트 인젝션 |
| 페르소나 디렉토리 외부 파일 접근 지시 | 중간 | 상위 디렉토리(`../`) 경로 참조 |
| 외부 URL 방문 또는 데이터 전송 지시 | 중간 | 외부 서버에 정보를 보내도록 지시 |
| 과도한 권한 요청 | 낮음 | 필요 이상의 파일 읽기/쓰기 요청 |

## 참고

- 이 검토는 정적 분석이다. 런타임 동작은 보장하지 않는다.
- 의심스러운 패턴이 있으면 사용자에게 구체적인 위치와 내용을 알리고 판단을 맡긴다.
- 페르소나 제작자를 신뢰하는 경우라도 임포트 후 검토하는 습관을 권장한다.
